# Security Policy

## Reporting a vulnerability

If you find a security issue in this project, please report it privately —
do **not** open a public issue.

- **Preferred:** [GitHub private vulnerability reporting](https://github.com/ryanflynn9299/personal-site/security/advisories/new)
- Or use the contact form on the site with a note that it is security-related
  (do not include exploit details in the first message).

You can expect an acknowledgement within a few days. Please allow time for a
fix to ship before any public disclosure.

## Scope

- This repository (Next.js frontend, server actions, middleware, CI, Docker
  configuration).
- Vulnerabilities in third-party services (Directus, Matomo) should be
  reported upstream, but misconfigurations of them in this repo are in scope.

## Internal documentation

Security posture, threat model, and secure development standards for
contributors live in [.docs/SECURITY.md](.docs/SECURITY.md).
